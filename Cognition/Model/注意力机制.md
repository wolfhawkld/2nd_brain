# 注意力机制

## 概念

注意力机制让模型能够动态关注输入的不同部分，根据相关性分配权重。突破了RNN的序列依赖限制，是Transformer的核心创新。

## 核心思想

```
Attention(Q, K, V) = softmax(QK^T / √d_k) * V
```

- **Q (Query)**: 查询向量，"我在找什么"
- **K (Key)**: 键向量，"我是什么特征"
- **V (Value)**: 值向量，"我的内容是什么"
- **注意力权重**: Q与K的相似度，归一化后作为V的权重

## 注意力类型

### Self-Attention (自注意力)
- Q、K、V都来自同一输入序列
- 每个位置可以关注序列中所有位置
- **应用**: Transformer编码器、BERT

### Cross-Attention (交叉注意力)
- Q来自一个序列，K、V来自另一个序列
- 实现跨序列信息融合
- **应用**: Encoder-Decoder注意力、多模态融合

### Multi-Head Attention (多头注意力)
```
MultiHead(Q,K,V) = Concat(head_1,...,head_h) * W^O
where head_i = Attention(QW_i^Q, KW_i^K, VW_i^V)
```
- 多个注意力头并行计算，关注不同子空间
- **应用**: Transformer标配

### Masked Self-Attention
- 通过掩码阻止关注未来位置
- **应用**: GPT等自回归模型

## 注意力变体

### 稀疏注意力
- 只关注局部窗口或特定位置
- 降低O(n²)复杂度
- **例子**: Longformer, BigBird

### 线性注意力
- 通过核函数近似softmax
- 复杂度降至O(n)
- **例子**: Linear Transformer, Performer

### Flash Attention
- IO感知的精确注意力算法
- 显存高效、计算加速
- **应用**: 大模型训练标配

## 注意力权重可视化

- 可解释性: 权重矩阵可视化展示模型关注点
- 常用于分析模型行为、调试

## 计算复杂度

| 类型 | 时间复杂度 | 空间复杂度 |
|------|-----------|-----------|
| 标准注意力 | O(n²d) | O(n²) |
| 稀疏注意力 | O(n·w·d) | O(n·w) |
| 线性注意力 | O(nd²) | O(nd) |
| Flash Attention | O(n²d) | O(n) |

*n: 序列长度, d: 特征维度, w: 窗口大小*
