# 注意力机制（Attention Mechanism）

> 模拟人类注意力的算法，计算两个向量的相关程度，让模型聚焦于关键信息。是 **Transformer 的核心创新**。

---

## 背景与直觉

**人类注意力**：在复杂环境中，人类能快速聚焦于重要物体（猎物、天敌），而非无差别处理所有信息。

**机器注意力**：
- 输入：大量信息
- 输出：加权后的关键信息
- 核心思想：不同信息给予不同权重

---

## 从 Attention 到 Self-Attention

### Attention（传统注意力）

用于 Seq2Seq 模型（如机器翻译）：
- Encoder 输出所有信息
- Decoder 决定把注意力放在哪些信息上
- 权重参数是**固定的**（可学习但与输入无关）

### Self-Attention（自注意力）

Transformer 的核心创新：
- 输入序列内部的注意力
- 权重参数由**输入决定**（不同输入 → 不同权重）
- 每个元素与序列中所有其他元素建立关联

---

## Self-Attention 数学原理

### Q、K、V 三元组

输入序列 X = [x₁, x₂, ..., xₙ]，通过三个线性变换：

```
Q = W_q X  →  Query（查询向量）：我要找什么？
K = W_k X  →  Key（键向量）：我有什么特征？
V = W_v X  →  Value（值向量）：我的内容是什么？
```

### 注意力计算

**Step 1: 计算相似度**
```
A = Q K^T  →  每个Query与每个Key的点积
```

**Step 2: 缩放**
```
A = A / sqrt(d_k)  →  防止点积过大
```
d_k 是 Key 的维度。

**Step 3: Softmax**
```
Attention_weights = softmax(A)
```
得到每个位置对其他位置的注意力权重。

**Step 4: 加权求和**
```
Output = Attention_weights V
```

### 完整公式

```
Attention(Q, K, V) = softmax(Q K^T / sqrt(d_k)) V
```

---

## Scaled Dot-Product Attention

为什么需要缩放？

```
问题：当 d_k 很大时，点积结果会很大
后果：softmax 的梯度趋近于 0（梯度消失）
解决：除以 sqrt(d_k) 使方差稳定
```

---

## Multi-Head Attention

### 为什么需要多头？

类比 CNN 的多个卷积核：
- 一个 head 捕捉一种相关性
- 多个 head 捕捉多种不同的交互信息

### 计算过程

```
对于 M 个 head：
1. 每个 head 有独立的 W_q^i, W_k^i, W_v^i
2. 生成 M 组 Q_i, K_i, V_i
3. 计算 M 个注意力输出 B_i
4. Concat 后通过线性层：
   Output = W_o [B_1; B_2; ...; B_M]
```

### 实现方式

| 方法 | 说明 |
|------|------|
| Narrow | 把输入切成 h 块，每块用一次注意力（快但效果差） |
| Wide | 对输入独立用 h 次注意力（效果好但更耗资源） |

## 与 CNN、RNN 的对比

| 特性 | CNN | RNN | Self-Attention |
|------|-----|-----|----------------|
| 计算复杂度 | O(k·n·d²) | O(n·d²) | O(n²·d) |
| 序列操作数 | O(1) | O(n) | O(1) |
| 最大路径长度 | O(log_k(n)) | O(n) | O(1) |
| 并行化 | ✅ 强 | ❌ 弱 | ✅ 强 |
| 长距离依赖 | ⚠️ 需深层 | ❌ 信息遗忘 | ✅ 直接连接 |

**Self-Attention 优势**：
- 并行计算（训练效率高）
- 直接连接任意位置（无信息遗忘）
- 最大路径长度 = 1

**Self-Attention 劣势**：
- 计算复杂度 O(n²)（长序列成本高）

## Positional Encoding

Self-Attention 不包含位置信息（打乱序列不影响输出）。

### Transformer 的方案

```
PE(pos, 2i)   = sin(pos / 10000^(2i/d))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d))
```

- pos：位置
- i：维度索引
- d：词嵌入维度

**特点**：
- 每个维度是不同频率的正弦波
- 可以通过线性变换得到相对位置
- 与词嵌入相加（等价于 concat）

---

## Masked Attention

用于 Decoder，防止"看到未来"：

### Padding Mask

处理不定长序列：
- 区分有效 token 和 padding
- padding 位置的 attention 权重设为 0

### Subsequence Mask

自回归生成：
- 只能看到之前的 token
- 未来位置 mask 为 -∞（softmax 后为 0）

---

## Transformer 架构

```
Encoder:
├─ Input Embedding + Positional Encoding
├─ Multi-Head Self-Attention
├─ Add & Norm (残差连接 + LayerNorm)
└─ Feed Forward + Add & Norm

Decoder:
├─ Output Embedding + Positional Encoding
├─ Masked Multi-Head Self-Attention
├─ Add & Norm
├─ Multi-Head Attention (Q来自Decoder, K/V来自Encoder)
├─ Add & Norm
└─ Feed Forward + Add & Norm
```

## Python 实现（简化版）

```python
import torch
import torch.nn.functional as F

def scaled_dot_product_attention(Q, K, V, mask=None):
    """
    Q: (batch, seq_len, d_k)
    K: (batch, seq_len, d_k)
    V: (batch, seq_len, d_v)
    """
    d_k = Q.size(-1)
    
    # 1. 计算相似度并缩放
    scores = torch.matmul(Q, K.transpose(-2, -1)) / torch.sqrt(torch.tensor(d_k))
    
    # 2. Mask（可选）
    if mask is not None:
        scores = scores.masked_fill(mask == 0, -1e9)
    
    # 3. Softmax
    attention_weights = F.softmax(scores, dim=-1)
    
    # 4. 加权求和
    output = torch.matmul(attention_weights, V)
    
    return output, attention_weights

class MultiHeadAttention(torch.nn.Module):
    def __init__(self, d_model, num_heads):
        super().__init__()
        self.d_k = d_model // num_heads
        self.num_heads = num_heads
        
        self.W_q = torch.nn.Linear(d_model, d_model)
        self.W_k = torch.nn.Linear(d_model, d_model)
        self.W_v = torch.nn.Linear(d_model, d_model)
        self.W_o = torch.nn.Linear(d_model, d_model)
    
    def forward(self, Q, K, V, mask=None):
        batch_size = Q.size(0)
        
        # 线性变换并拆分成多头
        Q = self.W_q(Q).view(batch_size, -1, self.num_heads, self.d_k).transpose(1, 2)
        K = self.W_k(K).view(batch_size, -1, self.num_heads, self.d_k).transpose(1, 2)
        V = self.W_v(V).view(batch_size, -1, self.num_heads, self.d_k).transpose(1, 2)
        
        # 注意力计算
        output, weights = scaled_dot_product_attention(Q, K, V, mask)
        
        # 合并多头
        output = output.transpose(1, 2).contiguous().view(batch_size, -1, self.num_heads * self.d_k)
        
        # 最终线性变换
        output = self.W_o(output)
        
        return output, weights
```

## Attention 的变体

| 变体 | 说明 |
|------|------|
| **Cross-Attention** | Q 来自一个序列，K/V 来自另一个序列（Decoder 中使用） |
| **Sparse Attention** | 只关注部分位置，降低复杂度 |
| **Linear Attention** | 用核函数近似，复杂度 O(n) |
| **Flash Attention** | GPU 内存优化，减少 HBM 访问 |
| **Grouped Query Attention** | 多个 Q 共享 K/V（LLM 常用） |
