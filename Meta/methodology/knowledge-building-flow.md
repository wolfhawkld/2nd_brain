# 知识构建流程

2nd Brain 的标准化知识生产流程。

## 流程步骤

### 1. 主张提出
Damon 提出一个待验证的认知/技能/记录/元认知/探索项。

形式：初步主张，可能包含主观判断。

### 2. 验证与补完
Outis 职责：
- 识别主张是否客观正确
- 分析适用范围与边界条件
- 补充遗漏的视角或反例
- 给出明确的结论

### 3. 分类定位
根据结论，识别该主张应放入：
- **Cognition** — 已验证的知识、概念、认知
- **Skill** — 可操作的方法、技能、流程
- **Memo** — 记录、备查、闪念
- **Meta** — 关于知识本身的知识
- **Horizon** — 待探索、实验性、未定论

子分类：在现有子分类中选择，或建议新建。

### 4. 生成双链内容
- 创建 `.md` 文档
- 使用 `[[link]]` 建立与相关笔记的双向链接
- 更新父级 MOC（Map of Content）
- Git commit

### 5. 延伸建议
生成后，Outis 可建议：
- 相关但尚未创建的知识点
- 需要补充的背景知识
- 可进一步深入的子话题

由 Damon 决定：
- 立即补充
- 稍后自行了解后再补充

## 流程示意

```
Damon 主张
    ↓
Outis 验证 + 补完
    ↓
确定分类位置
    ↓
生成双链文档 → Git commit
    ↓
建议延伸知识点（可选）
    ↓
Damon 决定是否补充
```

## 设计原则

- **验证优先**：不正确的认知不入库，或标注争议
- **双链连接**：知识孤岛无价值，必须建立关联
- **渐进构建**：不强求一次完美，允许迭代完善
